Módulo 1 - HTML5 | CSS

Conteúdo das aulas

  • Capítulo 3

    • Aula 1 - A diferença entre HTML, CSS e JavaScript

      • HTML: Hypertext markup language é uma linguagem de marcação de texto focada no conteúdo (tabelas, textos, imagens, listas, expressões etc) do código.

      CSS: Cascading style sheets, ou folhas de estilo em cascata, é a linguagem responsável pelo design do site. CSS significa que uma página vai modificando as características visuais (cores,tipografia, diagramação, etc) das outras.

      JavaScript: É uma linguagem de programação responsável pelas interações (menus, animações, popups, etc) do site.

  • Capítulo 6

    • Aula 4 - A tag img em HTML5

      • A tag img, vem acompanhada de "scr=", onde irá inserir o caminho da imagem (pode ser uma URL ou imagem do próprio diretório) e "alt=", onde é recomendável colocar do que a imagem se trata, para fins de acessibilidade de deficientes visuais que forem inspecionar o site.

      Exemplo:

      <img scr="https://cdn-icons-png.flaticon.com/128/3613/3613825.png" alt="ícone de comida"<p>

      *Obs: Para colocar imagem de diretório próprio, é necessário que esta imagem esteja localizada na mesma pasta do código html.

    • Aula 5 - Como mudar o favicon de um site

      • O favicon do site, é o ícone que aparece na guia de navegação. Ele poderá ser inserido com uma imagem de extensão .ico, dentro do cabeçalho do código HTML. Para isso é necessário acrescentar a tag "link: favicon" e selecionar o arquivo .ico escolhido para o site. No exemplo abaixo, o favicon desta página é o html.ico, inserido dentro do atributo "href". No atributo seguinte (type), é necessário colocar a extensão do arquivo, que neste caso é o "image/x-icon"

      Exemplo:

      <!DOCTYPE html>
      <html lang="pt-br">
      <head>
             <meta charset="UTF-8">
             <meta name="viewport"
      content="width=device-width, initial-scale 1.0">
      <title>NOME QUE APARECE NA ABA DE NAVEGAÇÃO</title>
      <link rel="shortcut icon" href="html.ico" type="image/x-icon"
      </head>

      *Obs: Para colocar o favicon, é necessário que este esteja localizado na mesma pasta do código html.

    • Aula 4 - Citações e Códigos

      • Para citações simples, geralmente utilizada no meio de um parágrafo, sem deslocamento, utiliza-se a tag <q>.

      Exemplo de citação simples:

      Como dizia minha avó: É melhor um pássaro na mão do que dois voando.

      Já para citações mais longas e que necessitem de um parágrafo só para si, podendo ou não ganhar deslocamento, utilizamos a tag <blockquote>, que pode ser acompanhada do parâmetro cite"", onde podemos inserir de onde aquela citação vem.

      Exemplo de citação em bloco:

      That's the thing about unhappiness, all it takes
      is for something worse to came along and you
      realize it was actually happiness after all
      Queen Elizabeth II


      Existe uma tag que faz com que o usuário ao passar o cursor sobre determinada palavra ou termo, apareça um pequeno texto com seu significado. É comumente utilizada para deixar claro ao usuário e aos mecanismos de busca o significado destes termos. Usamos, neste caso, a tag <abbr>, que virá acompanhada do parâmetro 'title=""', onde inserimos o significado da abreviatura.

      Exemplo:

      Curso de HTML e CSS para desenvolvedores Front-End.


      Existe ainda uma tag em para delimitação de códigos de diversas linguagens de programação, a tag <code>, que deixa as letras mono-espaçadas, facilitando na leitura do código.

      Exemplo sem a tag <pre>:

      <h1>Código em Python</h1>

      <code>

      num = int(input('Digite um número')) if num % == 0: print(f'O número {num} é par') else: print(f'O número {num} é ímpar') print('FIM DO PROGRAMA')

      </code>

      Obs: no exemplo acima, a formatação do código aparece no navegador sem formatação. Para solucionar este problema, utilizamos a tag <pre> dentro da tag <code>

      Exemplo com a tag <pre>: 

      
                <h1>Código em Python</h1>
                <pre><code>
                num = int(input('Digite um número')) 
                if num % 2 == 0: 
                    print(f'O número {num} é par)
                else:
                    print(f'O número {num} é ímpar')
                print('FIM DO PROGRAMA')
                </code></pre>

      Obs: no exemplo acima, com o uso da tag <pre>, o código obedece uma pre-formatação de texto, se guiando por quebras de linhas, espaçamentos e parágrafos.

  • Capítulo 9

    • Aula 1 - Listas <ol> e <ul>

      • As listas ordenadas, são aquelas onde a ordem dos itens é importante. Para este tipo de lista, usamos a tag <ol> para delimitar a lista dentro desta tag, usamos <li> para identificar cada item.

      Exemplo de lista ordenada:

        Minhas comidas preferidas:

      1. Macarrão
      2. Burrito
      3. Sushi
      4. Angu

      Obs: A tag <ol> possui um parâmetro type, que serve para configurar o tipo de marcador da lista.
      (1, A, a, I, i.)
      Além disso, é possível determinar o início da contagem da lista com o parâmetro start.
      (Por exemplo: <ol type "i" start "5">

      As listas não ordenadas são aquelas em que a ordem dos itens não interfere no seu significado. Comumente utilizada para organizar itens que não apresentam, necessariamente, uma classificação. Para este tipo de lista, usamos a tag <ul>.

      Exemplo de lista não ordenada:

      Lista de compras:

      • Chocolate
      • Biscoito
      • Chips
      • Água

      Obs: Na tag <ul>, podemos determinar o marcador da lista através da propriedade type, com os valores:
      disc - circle - square.

      Outra possibilidade de uso das tags <ol> e <ul>, são as chamadas listas mistas.

    • Aula 2 - Listas mistas e de definição

      • As listas de definições são aquelas que possuem termos e suas descrições. Estas listas não possuem marcadores.

      Toda lista de definição está dentro de uma tag <dl> (definition list), cada termo é um <dt> (definition term), e cada descrição é um <dd> (definition description).

      Exemplo de listas de definições: 

              <dl>
            <dt>HTML</dt>
            <dd>Linguagem de marcação para criação de conteúdo de sites.</dd>
                        
            <dt>CSS</dt>
            <dd>Linguagem de marcação para especificação de estilos em sites.</dd>
                        
            <dt>JavaScript</dt>
            <dd>Linguagem de programação para criar interações em sites.</dd>
        </dl>

      ** O resultado visual no navegador para o exemplo de lista de definições acima é:

      HTML
      Linguagem de marcação para criar o conteúdo de site.
      CSS
      Linguagem de marcação para a especificação de estilos em sites.
      JavaScript
      Linguagem de progrmação para criar interações em sites.

  • Capítulo 10

    • Aula 1 - Links e Âncoras em HTML5

      • Conhecidos como âncoras, os hyperlinks permitem que os sites estabeleçam conexões com outras páginas. Seja para acessar um novo site ou até para fazer download de um arquivo. Para criar um hyperlink, utilizamos a tag <a>. Esta tag contém um atributo, o href, que serve para criar uma referência de hipertexto, ou seja, onde colocamos a URL de um site.

      Exemplo de estrutura da tag <a>:

                  <h1>Criando um link</h1>
            <a href="https://arquituario.netlify.app/glossario/">Acesse o Arquituário.</a>

      Outras observações da tag <a>:

      • Podemos usar o atributo hreflang="" para indicar o idioma principal do site para onde o link está desviando o fluxo de navegação;
      • Para poder controlar a abertura de destino do site, utilizamos o atributo target com 4 valores, mas com destaque para:
        • _blank, para abrir o link em uma nova janela/guia;
        • _self, para abrir o link na janela/guia atual (padrão)
      • Outro atributo usado junto a tag <a> é o rel, que indica qual é a natureza do destino do hyperlink.
        Ele aceita 5 diferentes valores:
        • next, para links que vão para a página seguinte do site atual;
        • prev, para links que voltam para a página anterior do site atual;
        • author, para links para site do autor do artigo do site atual;
        • external, para links que não fazem parte do site atual;
        • nofollow, para links pagos;

      Obs: para links internos (ou locais), é fundamental que o documento .html esteja na mesma pasta. Para links externos, utilizamos a url.

      Exemplo: 

                  <a href="pag002.html" target="_self">
                Continuar navegando no site
            </a>
            <a href="https://arquituario.netlify.app/glossario/" target="_blank">
                Abrir em uma nova aba
            </a>

    • Aula 3 - Links para download

      • Para fazer download de algum arquivo no site, também utilizamos a tag <a>, mas acompanhada de outros atributos:

      • download: vai indicar o nome do arquivo a ser baixado;
      • type: vai indicar ao navegador o tipo de arquivo que será baixado;

      Exemplo: 

                  <a href:"arquivos/livro.pdf" download="livro.pdf" type="application/pdf"
            Baixe aqui o PDF do livro
            </a>
      NOTA: lembrando que o arquivo a ser indexado para download no site deverá estar na mesma página do código html.
      * O href indica onde está o arquivo.
      ** media types mais usados:

      • application/zip
      • text/html
      • text/css
      • text/javascript
      • video/mp4
      • video/H264
      • video/JPEG
      • audio/aac
      • audio/mpeg
      • font/ttf
      • image/jpeg
      • image/png

  • Capítulo 11

    • Aula 2 - Imagens que se adaptam sozinhas

      • Para adaptar as imagens do site aos diversos dispositivos, é necessário que criemos tamanhos diferentes dessas imagens. Nesta situação, devemos utilizar as tags <picture> e <source>.

      Exemplo de estrutura:

              <picture>
            <source media="(max-width: 1050px)" srcset="foto-m.png" type="image/png"
            <img src="foto-g.png" alt="texto alternativo">
        </picture>

      NOTA: A tag <source>, possui 3 atributos:

      • type: vai indicar o media type da imagem usada;
      • srcset: vai configurar o nome da imagem que será carregada quando o tamanho indicado for atingido;
      • media: vai indicar o tamaxo máximo a ser considerado para carregar a imagem indicada no atrivuto srcset

      * Outra observação acerca do <source>, é que sua ordem vai importar. Os itens mais acima devem ser as imagens de menor tamanho, ou seja, a ordem deverá ser crescente. De modo que o último item dentro da tag <picture> será a imagem padrão.

    • Aula 3 - Colocando áudio no site

      • É possível indexar áudios em sites utilizando o HTML. Basta usar a tag <audio> e dentro dela, o <source>.
      É importante destacar os principais atributos da tag <audio>, são eles:

      • preload: que vai permitir que o áudio seja precarregado ou não, aceitando os seguintes valores:
        • metadata: que vai permitir que o site carregue apenas as informações sobre o arquivo, como tamanho, tempo, informações de direitos autorais, etc;
        • none: que não vai carregar/reproduzir nada até que o usuário clique no botão de play;
        • auto: é o padrão do código, vai carregar o arquivo de áudio integralmente assim que a página for carregada, independente do usuário clicar no play;
          •
      • controls: vai mostrar o player na tela, permitindo a interação do usuário;
      • loop: faz com que o áudio se repita assim que sua reprodução é terminada;

      Exemplo de estrutura:

          <audio preload="metadata" controls autoplay lopp>
        <source src="midia/musica-004.mp3" type="audio/mpeg">
        <source src="midia/musica-004.mp3" type="audio/mpeg">
        <source src="midia/musica-004.mp3" type="audio/mpeg">
        <p>Seu navegador não suporta áudio
        <a href="midia/musica-004.mp3" 
        download="musica-004.mp3" type="audio/mpeg"></a></p>
    </audio>

      NOTA: assim como na dinamização de imagens, a ordem dos <source> vai importar. Sendo que na parte de cima devemos colocar o formato de áudio preferido, de modo que os demais só serão carregados caso os que estiverem acima dele não carreguem. Além disso, é possível criar uma outra alternativa para caso nenhum dos formatos indexados não carregarem, adicionamos um parágrafo que vai permitir que o usuário faça o download do áudio para ouvir no player padrão do dispositivo.

    • Aula 5 - Vídeos em hospedagem própria

      • Os vídeos de hospedagem própria funcionam da mesma forma que os áudios, mas usamos a tag <video> com outros atributos:

      • width: vai indicar a largura que o vídeo vai ter na tela;
      • poster: vai configurar a thumb que vai aparecer como capa do vídeo enquanto o usuário não der play;
      • controls: vai configurar se os controles do vídeo aparecerão na parte inferior da mídia, por padrão não vai aparecer, mas é só colocar este atributo;
      • autoplay: indica ao navegador se o vídeo vai reproduzir automaticamente assim que a página for carregada;

      Exemplo de estrutura:

              <video width="600" poster="thumb.jpg" controls autoplay>
            <source src="meu-video.webm"type="video/webm">
            <source src="meu-video.mp4" type="video/mp4">
            <source src="meu-video.ogv" type="video/ogg">
            <p>Infelizmente seu navegador não conseguiu carregar o vídeo.</p>
        </video>

    • Aula 6 - Incorporação de vídeos externos

      • A incorporação de vídeos externos é uma alternativa mais interessante e de menor custo, uma vez que estes vão consumir menos bytes por estarem hospedados em outros servidores. Desta forma, sites como Youtube e Vimeo, permitem que copiemos o código HTML dos vídeos desejados.

      Encontramos este código pronto no campo:

      Compartilhar > Incorporar > Copiar.

  • Capítulo 12

    • Aula 1 - Estilos CSS inline

      • O estilo CSS inline serve para alterações de estilos específicas e pontuais. Este tipo é caracterizado pela aplicação de estilos diretamente em um elemento HTML. Para isso, utilizamos o atributo style, que vai permitir alterações específicas em um único elemento sem precisar criar um documento (.css) separado.

      Exemplo: 

              <p style="color: blue;">Este é um parágrafo azul.</p>

    • Aula 2 - Estilos CSS internos

      • O estilo CSS interno (ou local) é aquele onde se inclui os estilos dentro do prórprio documento html, no cabeçalho (<head></head>).

      Exemplo: 

              <!DOCTYPE html>
        <html lang="pt-br">
        <head>
        <meta charset= "UTF-8">
        <meta name= "viewport" content="width=device-width, initial-scale= 1.0"
        <link rel="stylesheet"
            <style>
            p {
                    color:
                    font-size:
                    background-color:
                    font-family:
                }
            </style>
        </head>
        <body>
            <Este é um parágrafo com estilo interno.</p>
        </body>
        </html>

      NOTA: o uso do estilo CSS local é mais comum para sites menores, com poucas páginas. Nele, é possível fazer todas as configurações de estilo no mesmo documento HTML, o que pode deixá-lo mais longo.

    • Aula 3 - Estilos CSS externos

      • O estilo CSS externo é aquele em que criamos uma estrutura separada do documento html. Nele é possível definir os estilos a serem aplicados em todas as páginas que incluem o mesmo arquivo .css externo, possibilitando sites mais consistentes. Neste tipo de estrutura, quando o site necessitar de um estilo personalizado para cada página, podemos criar mais de um arquivo .css e incluir no documento html.

      Exemplo do arquivo .css dentro do código html:

              <!DOCTYPE html>
        <html lang="pt-br">
        <head>
            <meta charset= "UTF-8">
            <meta name= "viewport" content="width=device-width, initial-scale= 1.0"
            <link rel="stylesheet" type="text/css" href="style.css">
            <title>CSS externo</title>
        </head>
        <body>
            <h1>Estilo CSS externo.</h1>
        </body>
        </html>

      NOTA: nesse estilo, utilizamos a tag <link> dentro do cabeçalho para adicionarmos o documento .css, da mesma forma que se faz com o favicon. De forma que teremos os seguintes atributos:

      • rel: para indicar o vínculo da folha de estilo com o html;
      • type: para especificar o tipo de mídia;
      • href: que vai direcionar para o arquivo .css externo.
