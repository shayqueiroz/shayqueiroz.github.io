    • Aula 1 - Cores

      • Cores complementares: aquelas que têm maior contraste entre si, localizadas do lado imediatamente oposto do círculo cromático.

      Cores análogas: são aquelas que são imediatamente vizinhas entre si, apresentam baixo contraste.

      Cores triádicas: escolha de três cores equidistântes resultando em um ótimo contraste, e que formam um triângulo equilátero.

      Cores em quadrado: seleção de quatro cores com um contráste razoável entre as cores escolhidas.

      Cores tetrádicas: pares de cores complementares com bastante contraste entre si.

      Monocromia: variação do brilho e saturação de uma cor, modificando apenas seu tom.

    • Aula 6 - Criando degradê com CSS

      • A forma mais simples de criar um efeito degradê no site é usando o sistema rgb ou hsl, por fornecer mais opções de variação de cor. Ainda é possível adicionar transparência através do valor alpha, que virá sempre por último (caso queira diminuir a opacidade da cor).
      De forma prática, temos a seguinte estrutura em CSS para a criação de gradientes: 

                      body {
                    font-family:
                    background-image: linear-gradient (90deg, skyblue, deepskyblue);
                    color: black;
                }

      A linha em destaque corresponde a configuração do degradê em CSS. Na ordem, os valores correspondem a: direção/ângulo, uma primeira cor, uma segunda cor.

      Também existe outra forma de criar este efeito, usando ao invés de linear-gradient, que cria um degradê linear angular, o radial-gradient, que vai criar um efeito degradê de forma radial.

                      background-image: radial-gradient (circle, color 1, color 2, color 3 ...);

      NOTA: é possível escolher quantas cores quiser. O navegador automaticamente vai conseguir identificá-las e gerar o efeito degradê com as cores escolhidas.

    • Aula 2 - Anatomia do tipo

      • Categoria de fontes

      Existem cinco categorias para fontes tipográficas:
      • Serifadas: aquelas onde as letras possuem prolongamentos (serifas), que criam um efeito de leitura em bloco na nossa visão, guiando nossa leitua através das serifas.
      • Não Serifadas: não apresentam aquele prolongamento nas letras como as serifadas. São recomendáveis para sites devido a sua característica clean, organizada e clara.
      • Monoespaçadas: este tipo de fonte pode ou não apresentar serifas. São caracterizadas pelo espaço horizontal (largura) igual entre todas as letras.
      • Script: também conhecidas como handwriting, são fontes que buscam reproduzir a escrita humana.
      • Display: são aquelas que não se enquadram nas categorias acima, caracterizadas por efeitos visuais diferenciados e bastante gráficos.

    • Aula 3 - Família de fonte com CSS

      • Para atribuir uma família em um código CSS, utilizamos a propriedade font-family, que vai indicar ao navegador qual a fonte desejada para o conteúdo do site.

      Estrutura:

                      body {
                    font-family: 'Franklin Gothic Medium', 'Arial Narrow', Arial, sans-serif;
                }

      A estrutura acima indica uma ordem de preferência de fonte. Caso o navegador não reconheça a primeira, ele vai buscar as seguintes em ordem. Se ainda assim nenhuma das fontes forem encontradas ou compatíveis com o navegador, ele utilizará alguma outra com a categoria genérica para a categoria da fonte. Este recurso é chamado de safe-combination.

    • Aula 4 - Tamanho de fonte e suas medidas

      • Existem muitas formas de especificar medidas em CSS, mas a recomendação para tamanho de fontes, segundo a W3Schools são as unidades de px(pixels) ou em(emphasis)*.

      NOTA: * em está relacionada ao referencial do tamanho da fonte.

      Geralmente o tamanho padrão de uma fonte é de 16px, e isso equivale a 1em. Se quisermos uma fonte 2x maior do que a padrão, usaremos 2em, 3x maior, 3em e assim por diante. E isso corresponderá, em pixels, a 16px, 32px, etc respectivamente.

      Exemplo:

                      h1 {
                    font-family:
                    font-size: 2em;
                }
                h2 {
                    font-size: 1.5em
                }

      No exemplo acima, todo <h2> terá 1.5x o tamanho padrão da fonte de referência.

    • Aula 5 - Peso, estilo e shorthand font

      • Como CSS não é responsável pela semântica do conteúdo, é possível adicionar propriedades como negrito e itálico as fontes. Usamos para isso font-style e font-weight, respectivamente. Onde o font-style pode receber atribuição italic, normal ou oblique, e o font-weight variando de 500 a 900, ou bold, bolder, light, lighter e normal.

      NOTA: os valores de font-weight podem ou não ser compatíveis com o tipo de fonte. Existem fontes que não aceitam variações de peso.

      Shorthand-style

      É um atalho que permite reunir em um único lugar todas as configurações de fonte.

      Exemplo:

                      p {
                    font-family: Arial, Helvetica, sans-serif;
                    font-size: 1em;
                    font-style: italic;
                    font-weight: lighter;
                }

      Usando shorthand:

                      p {
                    font: italic lighter 1em Arial, Helvetica, sans-serif;
                }

      NOTA: é extremamente importante que a ordem da shorthand seja respeitada. No caso da propriedade font, a ordem é sempre:

      • font-style
      • font-variant
      • font-weight
      • font-size/line-height
      • font-family
        •

      Para alinhamento de texto utilizamos text-align com os seguintes valores:

      • left;
      • right;
      • center;
      • justify;

Para adicioná-las, é preciso escolher uma fonte, depois clicar em @import para ter acesso aos códigos que serão colocados no arquivo de estilos CSS. Esta parte do @import com as informações de url da fonte escolhida, deverá ser colocada na primeira linha da declaração do documento CSS, enquanto a especificação do CSS rules ficará no campo de propriedades do font-family de todo seletor que se desejar aplicar a fonte selecionada.

      Fontes externas

      OBS.: no font-family iremos dar nome para a fonte que baixamos. Uma vez que dermos nome a esta fonte, na linha seguinte, na parte de src: url, vamos selecionar onde está o arquivo dessa fonte. É importante, como forma de manter os arquivos organizados, criar uma pasta para os arquivos de fonte.
      Já no campo de format, temos 5 tipos:

      • opentype (.otf)
      • truetype (.ttf)
      • embedded-opentype
      • truetype-aat (Apple Advanced Typography)
      • svg
        •

      Mas os mais utilizados são os formatos opentype e truetype.

    • Aula 1 e 2 - Direfenças entre id e class em CSS

      Em HTML, quando queremos dar identidade para um elemento único, atribuimos a ele a propriedade id, e quando são mais de um elemento com características semelhantes, os agrupamos em class. É possível ainda, aplicar mais de uma classe a um elemento, e também é possível ter um id e uma class em um mesmo elemento, considerando que as propriedades de id irá sobrepor as propriedades da class.

      Seletores personalizados CSS

    • # = id
    • . = class
    • : = pseudo-class
    • :: = pseudo-element
    • > = children
      •

      Exemplo 1 - id e class no HTML:

                      <body>
                    <h1 id="título principal"> EXEMPLO DE TÍTULO </h1>
                    <h1> HTML e CSS </h1>
                    <h2 class="topico">SEMÂNTICA WEB </h2>
                    <p class="texto"> lorem ipsum lorem ipsum lorem ipsum lorem
                          ipsum lorem ipsum lorem ipsum lorem  ipsum lorem ipsum </p>
                </body>

      O <h1> principal possui um id, isso vai identificar um elemento único dentro da página atual e permitir criar um estilo especial para este elemento de forma isolada. Enquanto um class vai identificar um conjunto de um ou mais elementos (classe), que compartilham características comuns à todos da mesma classe.
      Em resumo, no exemplo acima, todos os elementos de <h1>, identificados pelo id receberam uma característica especial atribuída à ele, e todo elemento identificado com class="texto", receberam características especiais, assim como os de class="texto".

      NOTA: em um mesmo documento HTML, só podemos usar um id para um único elemento. Não é possível ter dois elementos com um mesmo id dentro de uma mesma página. Ao contrário do class, em que podemos ter vários elementos que possuam essa mesma característica.

      Exemplo 2 - id e class no CSS

                      body {
                    font: lighter 1em Arial, Helvetica, sans-serif;
                }
                h1 {
                    font-size: 2em;
                    color: deepskyblue;
                }
                h2 {
                    color: blue;
                }

      Exemplo 3 - seletores: id e class:

                      #titulo-principal {
                        background-color: deepskyblue;
                        color: white;
                        text-align: center;
                }
                .topico {
                        text-align: right;
                }
                .texto {
                        text-align: justify;
                }

      Exemplo 1, estamos identificando semanticamente quais elementos receberão características especiais (h1 terá um id e o h2 e p terão classes diferentes).
      No Exemplo 2, atribuimos características para os elementos do documento HTML, como o tipo da fonte para o corpo do site e o tamanho e cor para os títulos.
      No Exemplo 3, caracterizamos os seletores especiais (id e class). Atribuído anteriormente no documento HTML (Exemplo 1). Nele personalizamos para o título, a cor de fundo, a cor do texto e o seu alinhamento.

      Propriedades herdadas

      As propriedades herdadas referem-se as características que são transmitidas de um elemento pai para seus elementos filhos. Ou seja, quando aplicamos uma propriedade a um elemento principal, essa propriedade pode ser automaticamente aplicada aos elementos dentro dele, a menos que sejam especificamente sobrescritos.
      Isso significa dizer que, caso haja duplicidade nas configurações, o que fica valendo no final é a propriedade especificada nas configurações mais abaixo do código. Cada camada de aplicação de estilos vai adicionando propriedades novas e sobrescrevendo as configurações do nível anterior.

      Exemplo: Num documento HTML para aplicarmos estilos, temos a seguinte ordem:

      1. CSS externo
      2. CSS local
      3. CSS inline

      Com isso, temos que as especificações de estilo acima do inline, vão adicionar novas configurações de estilo, mas em caso de duplicidade (como já dito acima), o que prevalece será o que foi especificado nas configurações inline.

      NOTA: o uso do símbolo ">" em um seletor, indicamos os filhos imediatos de um elemento.

                      div:hover > p {}

    • Aula 3 - Pseudo-classes em CSS

      • As pseudo-classes são seletores especiais que permitem aplicar estilos a elementos específicos. Elas são identificadas por ":" e são usadas para selecionar elementos que estão em uma posição particular dentro do documento HTML.
      As pseudo-classes são bastante úteis para aplicar estilos a elementos interativos com base nas ações do usuário. Elas adicionam uma camada extra de flexibilidade ao estilo dos elementos, permitindo interações visuais no site.

      As pseudo-classes mais comuns são:

      • :hover - aplica estilo quando o cursor estiver sobre o elemento.
      • :visited - usado para mostrar que um link da página já foi visitado.
      • :active - usado para aplicar estilo a um elemento enquanto ele estiver sendo clicado ou pressionado.
      • :checked - usado junto com elementos de entrada como caixas de seleção, permitindo que aplique estilo aos elementos marcados ou selecionados pelo usuário.
      • :empty - usado para selecionar elementos que não têm nenhum conteúdo filho, incluindo elementos que não têm texto ou outros elementos aninhados dentro deles.
      • :focus - usado para selecionar um elemento quando ele está em foco, ou seja, quando o usuário interage com o elemento.

    • Aula 4 - Pseudo-elementos em CSS

      • Os pseudo-elementos seletores especiais que permitem adicionar estilos a partes específicas de um elemento, como se fossem elementos HTML independentes, mas sem a necessidade de adicionar marcação adicional. Os pseudo-elementos são representados por "::" seguidos pelo nome.

      Os pseudo-elementos mais comuns são:

      • ::before - permite adicionar conteúdo antes do conteúdo real do elemento. É frequentemente usado para adicionar elementos visuais, como ícones ou decorações.
      • ::after - permite adicionar conteúdo após o conteúdo real do elemento, atribuindo detalhes visuais, como linhas de separaçao ou marcadores de fim.
      • ::first-letter - seleciona a primeira letra de um elemento, permitindo aplicar estilos exclusivos a ela. É muito usado para criar um efeito de letra inicial decorativa.
      • ::first-line - seleciona a primeira linha de um elemento, como um parágrafo. Permitindo que se aplique estilos à primeira linha do texto, como alterar a fonte ou adicionar uma cor diferente.

    • Aula 1 a 3 - Modelando Caixas

      • As caixas em HTML são blocos de conteúdos. Estes blocos são como uma caixa retangular invisível. Quando criamos coisas em uma página web, como textos, imagens, vídeos etc, estamos colocando esses elementos em diferentes tipos de caixas que podem receber estilos e ser organizadas com CSS para formar o layout da página.

      Anatomia de uma caixa

      Toda caixa em HTML possui as seguintes propriedades:

      • content: área onde o conteúdo real do elemento é exibido. Esta área de conteúdo é conhecida como box-size, influenciada pela:
        • width: largura
        • height: altura
      • padding: área de preenchimento entre o conteúdo e a borda (da borda para dentro).
      • border: área que fica em volta do conteúdo, podendo receber definições de espessura, cor e formato.
      • margin: área externa da borda.

      Existe ainda o outline, que é a área entre a borda e a margem. É pouquíssimo utilizada, possui um traçado visual que podemos criar fora da borda, e o cálculo da sua espessua faz parte da margem estabelecida.

      NOTA: para sabermos a largura e altura total de um elemento na tela, devemos somar os tamanhos:

      content + padding + border + margin.

      Na imagem acima temos:

                      h1 {
                    width: 300px;
                    height: 50px;
                    background-color: lightpink;
                    border-width: 10px;
                    border-style: solid;
                    border-color: aqua;
                    padding: 20px;
                    outline-widith: 30px;
                    outline-style: solid;
                    outline-color: skyblue;
                    margin: 50px;
                }

      As medidas de height e width, 300px e 50px respectivamente, correspondem as medidas somente da parte pontilhada do conteúdo, onde está escrito "Exemplo de Caixa".

      A parte na cor aqua, que é a border, tem 10px e a parte skyblue outline 30px. O padding é de 20px e fica da borda para dentro, por fim margin é de 50px e fica da borda para fora.

      Deste modo, a medida total que essa caixa vai ocupar é: width = 460px e height = 210px.

      Shorthand

      Podemos simplificar as configurações de borda e contorno com uma shorthand. E a ordem deve ser sempre a mesma, tanto para as shorthands -width, -style, -color.

      Modo completo

      border-width: 10px;
      border-style: solid;
      border-color: yellow;
      outline-widith: 30px;
      outline-style: solid;
      outline-color: blue;

      Shorthand

      border: 10px solid yellow;
      outline: 30px solid blue;

      Padding e Margin

      Todo box-element tem quatro valores para padding e margin, e será considerado sempre os valores no sentido horário: -top, -right, -bottom, -left. Estas configurações também possuem shorthands.

      Modo completo

      padding-top: 10px;
      padding-right: 15px;
      padding-bottom: 20px;
      padding-left: 25px;
      margin-top: 0px;
      margin-right: 10px;
      margin-bottom: 20px;
      margin-left: 30px;

      Shorthand

      padding: 10px 15px 20px 25px;
      margin: 0px 10px 20px 30px;

      NOTA: quando os valores para padding e margin tiverem um único valor (ex.: 10px para top, right, bottom e left), podemos encurtar e usar padding:10px; | margin: 10px; que este valor será aplicado de forma simétrica para todas as direções.
      Para duas medidas iguais, sempre em pares (top-bottom e left-right), podemos usar o shorthand padding: 10px 20px; | margin: 0px 15px;

      Existe um recurso muito usado para personalização dos blocos, que são as margens automáticas. Este recurso é muito interessante pois permite que o navegador calcule automaticamente as margens laterais de forma que o bloco de conteúdo se posicione no meio, independente das dimensões da janela.
      Para isso, utilizamos a seguinte declaração no seletor:

                      margin: auto;

      Largura e altura adaptáveis

      Caso não façamos ajustes no width e heigth das caixas, esta vai ocupar 100% do bloco. Mas para que o site se adapte mais facilmente ao tamanho da tela, podemos deixar algumas caixas mais flexíveis indicando valores mínimos e máximos.

                      min-width: 400px;
                max-width: 800px;

      No exemplo de configuração acima, os valores vão garantir que a caixa não fique com largura menor que 400px (para telas menores) e nem seja maior que 800px (telas muito grandes).

      Para medidas entre 400px e 800px, o tamanho será adaptável e a caixa vai ocupar 100% da largura do bloco.

      Tipos de Caixas

      Existem duas categorias de caixas, a do tipo block-level e inline-level.

      Por padrão, a caixa do tipo block-level, apresenta os elementos posicionados como blocos independentes em relação aos outros. Eles começam em uma nova linha e ocupam a largura total do navegador. Já a caixa do tipo inline-level, os elementos não começam em uma nova linha. Eles fluem dentro do conteúdo ao redor, ocupando apenas o espaço necessário para exibir seu conteúdo.

      Na imagem acima temos exemplos de block-level e inline-level.

      Nestes dois modelos de caixa podemos destacar alguns exemplos de tags para cada um deles.

      Elementos HTML block-level

      <div> <nav> <main> <header> <footer> <h1>-<h6> <article> <aside> <ol> <ul> <li> <dt> <dd> <p>

      Elementos HTML inline-level

      <a> <abbr> <img> <em> <br> <strong>

    • Aula 4 - Grouping Tags em HTML5

      • Em HTML as tags são semânticas, enquanto em CSS atribuimos estilo a elas. Diante disso, as grouping tags surgem como tags semânticas de agrupamento. O que antes era feito de forma genérica com a tag <div>e a tag <span>, cuja diferença básica entre elas é ser do agrupador block-level e inline-level respectivamente. As tags semânticas de agrupamento servem para juntar vários outros elementos HTML.

      Agregadores semânticos da linguagem de HTML5

      • <header> - cria áreas relativas ao cabeçalho. Pode ser o cabeçalho principal de um site ou até mesmo um cabeçalho de uma seção ou artigo, podendo conter menus de navegação.
      • <nav> - define uma área que possui os links de navegação pela estrutura de páginas que vão compor o site, pode estar dentro de um <header>. Dentro do <nav>, podemos adicionar uma propriedade que permite uma animação do menu, para isso utiliza-se nav > a {transition-duration;}, além das demais configurações de cor, margin, padding, border, font-weight, text-decoration etc.
      • <main> - é um agrupador usado para delimitar o conteúdo principal do site, onde se concentra as sections, articles e asides..
      • <section> - cria seções para sua página, podendo conter o conteúdo diretamente no seuc corpo ou dividi-los em artigos com conteúdos específicos.
      • <article> - é um elemento que vai conter um conteúdo que pode ser lido de forma independente e diz respeito a um mesmo assunto.
      • <aside> - delimita um conteúdo periférico e complementar ao conteúdo principal de um artigo ou seção.
      • <footer> - cria um rodapé para o site inteiro, seção ou artigo, é um elemento que possui autoria do desenvolvimento do site.

    • Aula 5 - Sombras nas caixas

      • As box-shadows são usadas para adicionar um efeito de profundidade nos blocos de conteúdo do site. É importante dizer que o exagero nas suas configurações pode ser visualmente desagradável para o usuário que navega pelo site. Portanto é aconselhável que usemos com equilíbrio esta configuração.

      Podemos configurar as sombras de tal maneira:

                      article#caixa {
                    background-color: cinza;
                    width: 400px;
                    margin: auto;
                    padding: 1px 10px;
                    box-shadow: 3px 5px 4px black;
                }

      No código acima, temos um artigo em uma caixa centralizada, por conta do margin: auto;, de cor cinza, com padding de um 1px de espaçamento na parte superior e inferior, e 10px de espaçamento nas laterais direita e esquerda, com sombra de deslocamento horizontal (h-offset) de 3px, deslocamento vertical (v-offset) de 5px, embaçamento (blur) de 4px e a cor preta para sombra, que pode ser acrescida de transparência para dar um aspecto mais suave e natural pra caixa.

      NOTA: a ordem dessa propriedade será sempre nestes valores: h-offset - v-offset - blur - color.
      Considerando que para valores negativos de h-offset e v-offset, as sombras vão se deslocar para esquerda e para cima, respectivamente.

      Sombras em textos

      Assim como existe o box-shadow para sombrear as caixas, existe também a propriedade text-shadow, que permite que seja aplicada sombra no texto. Como no box-shadow, o text-shadow também possui valores que devem ser aplicados em ordem para h-offset, v-offset, blur, color. Essa propriedade serve para dar destaque na letra e no seu background, criando um contraste entre elas.

                          text-shadow: 2px 2px 0px rgba (0, 0, 0, 0.455);

      * O último valor do rgba no código acima, é relativo à transparência da sombra.

    • Aula 7 - Bordas decoradas

      • As bordas decoradas, também conhecidas como bordas estilizadas, são atribuídas quando queremos adicionar elementos especiais ao redor nas caixas de conteúdo. Podem ser ir de decorações mais simples, como pontos, traços e arredondamento de vértices até imagens e cores.

      Nas bordas decoradas, teremos 3 propriedades:

      border-image-source:(indica o caminho da imagem a ser usada como borda)
      border-image-slice: (indica quanto da imagem será usada para cada uma das bordas, aceitando até 4 valores: borda superior - borda direita - borda inferior - borda esquerda)
      border-image-repeat:(indica que a imagem vai se repetir ao longo da margem)

      NOTA: as configurações para bordas decoradas vem podem vir acompanhadas de outras propriedades, como text-align, padding, border etc.

    • Aula 6 - Variáveis em CSS

        • No documento .css (ou dentro da tag style, quando as configurações de estilo forem feitas no próprio documento html) criamos uma área de definições no cabeçalho para uma pseudo-classe "root {}", que vai permitir que todas as definições criadas dentro dela sirva para o documento inteiro, criando variáveis globais. Essas variáveis sempre vão começar com "--".

        Exemplo:

                        :root {
                    --cor0: #ccebd1;
                    --cor1: #daf5d5;
                    --cor2: #d5decc;
                    --cor3: #f1f5d5;
                    --cor4: #ebe9cc;
                    --fonte0: Roboto, Helvetica, sans-serif;
                    --fonte1: 'BB Manual Mono Pro TX', mono;
                    --fonte2: 'Outra Fonte', sans-serif;
                }

        A partir dessa pseudo-class :root {} do exemplo acima, é possível definir cores e fontes nos elementos HTML de forma mais simples e personalizável, usando a função var().

                        body {
                    font-family: var(--font0);
                    color: var(--cor0);
                    background-color: var(--cor1);
                }

        O seletor * em CSS, tem a função de seletor global. Ele será responsável por permitir que todas as configurações padrão de estilo definidas sejam atribuídas em todo documento, permitindo que se configure os elementos de forma manual.

                        * {  }

        Espaçamento entrelinhas em textos

        O espaçamento entre as linhas dos textos é muito importante, principalmente para texto muito longos. Para configurar isso, utilizamos a propriedade line-height, que vai permitir que possamos definir o tamanho do espaçamento entre uma linha e outra.

        Listas personalizadas

        É possível adicionar outros tipos de configurações de listas além das ordenadas <ol>, não ordenadas <ul> e mistas <ol> e <ul>. Uma dessas configurações é dividir uma lista em duas colunas e modificar o marcador para além dos marcadores padrão (disc, circle, square).

        NOTA: podemos usar marcadores que quisermos, desde que ele possua um código, como os do Emojipedia.

                        ul {
                    list-style-type: '\2611\0020\0020';
                    columns: 2;
                    list-style-position: inside;
                }

        No exemplo acima, os valores do primeiro parâmetro (list-style-type), corresponde a um símbolo de código U+2611 referenciado no Emojipedia. Já o valor \0020, que acompanha o código do emoji, corresponde a um espaço em branco. Já o list-style-position, vai fazer com que os marcadores sejam exibidos na parte interna da caixa que contém a lista, por padrão, este valor é outside.


    • Aula 7 e 8 - Responsividade

      • Sites responsivos são aqueles que se adaptam a vários dispositivos com tamanhos diferentes de tela. Isto é, o site vai ser capaz de adaptar os tamanhos dos seus componentes para manter o site visível e proporcionar uma experiência de usuário otimizada em qualquer tela.

      Para deixar um site responsivo, utilizamos o min-width e max-widith. Os valores para estas propriedades vão variar de acordo com o tipo de site, e devem ser testados para se adaptar ao que está sendo proposto.

      Configurações de imagem

      Para fazer ajustes nas imagens, fazemos modificações dos valores de img {width: x %;} ou max-width: 000px;. E no conteúdo HTML, usamos dentro da tag <picture> os <sources>, deixando a imagem padrão mais embaixo na ordem.

      Além disso, para centralizar imagens, podemos usar a seguinte configuração:

                      main img.pequena {
                    max-width: 350px;
                    display: block;
                    margin: auto;
                }

      Exemplo

      (Redimensione a janela do seu navegador para perceber a responsividade da imagem)

      Explicação

      Isso significa que a imagem de class ='pequena' foi ajustada para não perder muito da sua resolução, foi criado também um bloco para esta imagem (lembrando que os blocos possuem margin - padding - border), e por fim, por conta do margin: auto; a imagem ficará centralizada no site.

    • Aula 12 - Vídeos Responsivos

      • Assim como as imagens, também é possível adicionar configurações para que os vídeos se adaptem os mais diversos dispositivos.
      Quando utilizamos vídeos do Youtube ou Vimeo, por exemplo, eles já vêm com uma pré-configuração, o <iframe>, que possui dimensões fixas. Para que possamos alterar isso, de forma que o vídeo possa se adaptar melhor a interface do site ficando mais responsivo, precisamos colocar o <iframe> de incorporação dentro de uma <div>, de forma que o vídeo esteja limitado dentro de um bloco.

                      <div class="video">
                    <!-- CÓDIGO DO IFRAME EXTRAÍDO DO YOUTUBE/VIMEO -->
                </div>

      Depois disso, é recomendável fazer as configurações de estilo para os dois elementos:

                      div.video {
                    position:
                    background-color:
                    height:
                    margin-top:
                    margin-right:
                    margin-bottom:
                    margin-left:
                    padding-bottom:
                } 
                
                div.video > iframe {
                    position:
                    top:
                    left:
                    width:
                    height:
                }

      Exemplo

      Em resumo, na declaração HTML envelopamos o <iframe> dentro da tag <div> e criamos um id (caso o vídeo seja um elemento único) ou um <class> (em caso de mais de um elemento).
      Depois, no documento CSS, criamos o div.video {}, onde faremos as configurações de background, margin, padding, position, etc. Por fim, criamos o div.video > iframe {}, para fazer as configurações finais de deslocamento à esquerda e ao topo, seu tamanho em largura e altura (todos em porcentagem).

      Observações: na div para os valores de position:, se colocarmos relative, significa que será considerado o posicionamento atual do elemento de divisão, e ele vai se manter adaptável para caso de alteração das dimensões do navegador.
      Dentro do iframe, se usarmos o position: com valor absolute, a div, que é o bloco, vai se tornar o ponto de partida para o posicionamento do frame. A partir disso é possível fazer as configurações de deslocamento (top-left) e de tamanho (width-height).
